# CAD Auto IME 项目说明文档 (Markdown版)

**专注绘图，告别输入法打断**

彻底解决 CAD 绘图时中英文频繁手动切换的效率痛点。自动识别当前输入场景，精准、无感地完成输入法切换。

**支持版本:** AutoCAD 2007-2027 | **底层架构:** 高度解耦的 .NET 多目标编译架构 (全面兼容 .NET 2.0 ~ .NET 10.0)

## 🎯 痛点与效能革命

在传统 CAD 操作中，命令输入（需英文）与文字标注（需中文）的频繁切换，极易打断设计师的思路，导致效率大幅度降低。引入 CAD Auto IME 后，效能获得了质的飞跃：


| 📊 效能对比维度               | 🛑 传统手动切换 | ✨ CAD Auto IME 方案 | 💡 核心提升                                     |
|---------------------------------------|-------------------------|-------------------------|-------------------------------------------------------|
| **操作中断次数** (次 / 标注) | 8.5 次                 | **0.2 次**             | **↓ 97%**: 传统每次标注均需手动切换     |
| **平均动作耗时** (秒)          | 4.2 秒                 | **0.8 秒**             | **0 感知**: 插件自动识别，无缝衔接       |
| **心流维持度** (%)               | 45%                     | **98%**                 | **↑ 117%**: 连续绘图与注释效率大幅提升 |


## ⚡ 核心功能引擎

经历全新底层架构升级，探索驱动插件运转的硬核技术特性：

- **🧠 智能场景与焦点穿透**：不再依赖脆弱的表层焦点。引入系统级 `GetGUIThreadInfo` Win32 API，直接穿透 AutoCAD 复杂的嵌套 UI（包含悬浮命令行、动态输入框），精准捕获真实输入光标，实现 100% 准确的输入法状态切换。
- **🕸️ 自愈型生命周期事件网**：彻底解决多开 / 新建图纸时的监听失效问题。插件会在文档切换（DocumentBecameCurrent）瞬间进行状态的二次校验与事件动态补挂，无论你怎么拖拽、多开图纸，监听器始终在线。
- **⏱️ 微秒级异步引擎防假死**：引入 ThreadPool 异步延迟机制，完美规避 CAD 弹出文字编辑框（如 MTEXT）时的焦点时差问题。配合 `ImmSetConversionStatus` 强制底层模式覆写，**彻底消灭切换后依然输入大写英文字母的终极 Bug**。
- **👻 无感后台与首帧重置**：启动 CAD 自动静默加载。每次切换图纸的 "首帧" 将静默下达最高优先级的英文重置指令，确保双手摸到键盘的第一下绝对是干净的纯英文环境。
- **🎛️ 现代化可视化配置面板**：纯原生 WinForms 暗黑主题界面，**16px 大圆角无边框设计**，Win32 API 实现窗体拖拽与暗黑滚动条。内置完整的 **Ctrl+Z 撤销 / Ctrl+Y 重做 / Delete 删除** 快捷键体系，实时搜索过滤。
- **💾 原子化防死锁配置存储**：配置不仅云端化漫游保存，更引入了原生的**安全 Copy 覆盖机制**，辅以多进程失败重试机制，彻底终结双开 CAD 或杀毒软件扫描时造成的配置清空、死锁等致命隐患。

## 🚀 使用与开发指南

### 🧑‍💻 最终用户指南

**📌 安装与激活步骤**

1. 前往仓库 Releases 页面，下载最新版本安装包。
2. **关键：关闭所有正在运行的 AutoCAD 窗口。**
3. 运行安装程序，按照向导完成底层注册表注入。
4. 启动 AutoCAD，底部命令行出现右下角黑透渐变欢迎悬浮窗即为成功生效。


>
> ⚠️ 防呆提示：请务必先安装并运行过至少一次 AutoCAD 再安装本插件。若加载后提示「不受信任」，请将安装目录添加至 CAD 的 TRUSTEDPATHS 变量中。
>

**⌨️ 常用控制命令**

- CUSTOMAUTOIME：打开可视化面板，配置白名单。
- TOGGLEAUTOIME：一键开启 / 关闭自动切换功能。

### 🛠️ 二次开发编译 (Shared Project 架构)

本项目采用高度解耦的 "共享项目 (.shproj)" 架构。以 `OpenCadIme.Core` 作为业务核心，通过 11 个不同的目标项目 (Sys17 到 Sys27) 隔离编译，实现一份核心代码横跨 AutoCAD 2007~2027 多个版本的壮举。

**编译本地环境准备步骤：**

```bash
# 1. 克隆代码库          
git clone https://github.com/.../CADAutoIME.git          
          
# 2. 补充受版权保护的依赖库 (极重要)          
mkdir lib          
# 前往本地 AutoCAD 安装目录，提取以下文件：          
# acmgd.dll, acdbmgd.dll, accoremgd.dll          
# 放入对应的 lib/SysXX 子文件夹中。          
          
# 3. 编译          
# 使用 Visual Studio 2026 打开 OpenCadIme.sln，Release 模式生成。
```

## 📅 历史版本与更新日志

### v0.2 架构重构版

*(底层架构跨越式升级)*

- **UI 焕新**：纯原生 WinForms 暗黑主题界面，16px 大圆角无边框设计。
- **生命周期自愈**：彻底解决多开 / 新建图纸时的监听失效问题，引入 DocumentBecameCurrent 事件动态补挂。
- **异步防假死引擎**：引入 ThreadPool 异步延迟机制，规避 MTEXT 文字编辑框焦点时差。

#### v0.2.1 (Hotfix Update)

- **【核心】双重控制引擎**：重写 `ImeController`，引入电源开关与中英文模式 (ConversionStatus) 双重控制，彻底解决 Win10/Win11 下需手动按 Shift 切换中英文的痛点。
- **【修复】TSF 架构降维打击**：加入 `ImmGetDefaultIMEWnd` 消息机制，修复 Windows 10 教育版及特定第三方输入法下，常规 API 切换失效的终极 Bug。
- **【稳定】线程并发锁**：为 `PluginMain` 引入 `_docLock` 线程锁，修复多图纸并发或极高频操作时引发的 AutoCAD"集合已被修改 (Fatal Error)" 致命崩溃。
- **【安全】配置防空锁**：升级 `ConfigManager`，废弃高危的 Delete+Move 操作，改用原生的文件 Copy 覆盖机制，彻底消除多进程抢占下白名单被意外清空的隐患。
