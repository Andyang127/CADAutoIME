>
> **AutoCAD 输入法自动切换插件程序**
>
> “让每一位绘图者都能专注于创作，而非输入法切换”
>

>

根据命令类型自动切换中英文输入法状态，彻底消除手动切换的操作中断；

>

>

- **版本**: v0.4.0
- **命名空间**: `OpenCadIme`
- **核心能力**: 智能切换 | 双引擎判定 | 跨代兼容 | 多文档隔离

---

## ✨ 功能特性

- **智能自动切换**：执行文字类命令时自动切中文，执行绘图类命令时自动切英文。
- **双引擎判定**：结合「命令事件监听」与「窗口焦点监测」，将误切率降至极低。
- **多重保险懒加载**：v0.4.0 全新 Anti-Freeze 引擎，彻底解决老版本 CAD 秒进报错、高版本 CAD 启动死锁问题。
- **跨代 UI 引擎**：WinForms与WPF界面，资源 `AssemblyResolve` 内存导航技术，2007~2027 全系 CAD 配置面板秒开无崩溃。
- **多文档状态隔离**：每张图纸维护独立的输入法状态，多开窗口作业互不干扰。
- **可视化白名单配置**：支持自定义添加命令的图形化管理界面，支持撤销/重做。
- **隐式命令兼容**：支持双击编辑文字（`_TEXTEDIT` 等隐式命令）自动切换。
- **防打扰机制**：智能区分系统后台唤醒与用户手动敲击命令，手工操作面板时不弹欢迎窗。
- **开箱即用**：内置 98 个常用命令（47 个原生 + 51 个第三方），完美覆盖主流插件。
- **极致轻量运行**：内存复用 + 防抖优化，对 CAD 宿主性能影响极小。

### 基本使用

安装后无需额外配置即可丝滑使用。插件会自动识别当前执行的命令类型，切换对应的输入法状态。

**常用控制命令：**


| 命令          | 功能                                      |
|:----------------|:--------------------------------------------|
| `CUSTOMAUTOIME` | 打开白名单配置面板                 |
| `TOGGLEAUTOIME` | 一键全局开启/关闭自动切换功能 |

## ⚙️ 白名单配置指南

### 配置面板操作

在 CAD 命令行输入 `CUSTOMAUTOIME` 打开配置面板，支持以下操作：

- **添加命令**：输入命令全称，回车或点击添加。
- **批量添加**：支持空格、逗号、分号分隔的批量输入。
- **删除命令**：双击条目或选中后按 Delete 键。
- **搜索过滤**：顶部搜索框实时过滤。
- **撤销/重做**：Ctrl+Z 撤销，Ctrl+Y 重做。
- **恢复默认**：一键恢复系统内置的默认命令。

>
> ⚠️ **重要提示**
>

>
> 添加命令时必须输入**命令全称**（Full Command
> Name），不能使用自定义的快捷键（Alias）。
>
>
> - ✅ 正确：`BLOCK`, `CIRCLE`
> - ❌ 错误：`B`, `C`
>

### 配置文件位置

配置文件默认安全存储于：

```text
C:\Users\用户名\AppData\Roaming\OpenCadIme\AutoImeCommands.txt
```

配置文件为纯文本格式，每行一个命令，支持`//`开头的注释行。进阶用户也可直接编辑该文件，重启 CAD 后立即生效。

## 🧩 工作原理

### 双引擎判定机制

插件采用两种机制结合的方式，精准判断当前是否需要切换为中文：

1. **命令事件引擎**：监听 `CommandWillStart` 和 `CommandEnded` 事件，当执行白名单内的文字命令时，标记“文本命令激活”状态。
2. **焦点雷达引擎**：通过 `SetWinEventHook` 监听 `EVENT_OBJECT_FOCUS` 事件，实时捕获当前焦点窗口的类名。


**互补逻辑：**

- **命令事件引擎**负责决定“什么时候该切中文”。
- **焦点雷达引擎**负责决定“什么时候该切回英文”（例如右键菜单、智能提示弹窗抢夺焦点时强保英文状态）。

### 多重保险懒加载（v0.4.0 新增）

1、引入 Anti-Freeze 懒加载引擎：

- 摒弃在 `IExtensionApplication.Initialize` 中同步初始化全部组件的做法。
- 监听 `DocumentCreated` / `DocumentActivated` 事件，待图纸 100% 就绪后再安全切入系统。
- 多重超时保护，极端情况下自动回退到最小功能集，绝不导致 CAD 启动卡死。

2、彻底解决高版本 CAD WPF 资源加载崩溃的历史难题：

- 订阅 `AppDomain.AssemblyResolve` 事件，实现内存级别的程序集安全寻路。
- 智能适配 MFC 消息循环（老版本）与 WPF/XAML 架构（新版本）。
- 动态物理隔离挂载，确保 2007~2027 全系 CAD 统一安全的调用入口。

## ❓ 常见问题 (FAQ)

\*\*Q1：为什么有些命令不自动切中文？\*\*A：请排查以下几点：

1. 确认输入的是命令全称，而非快捷键。
2. 确认该命令已添加到配置面板的白名单中。
3. 部分插件的命令若通过非标准方式（如强行劫持菜单）触发，可能无法被底层的命令事件捕获。
4. 双击编辑文字等场景，请务必确认已升级至 v0.4.0 及以上版本。


\*\*Q2：可以删除 TEXT、MTEXT等系统内置命令吗？\*\*A：强烈建议不要删除。这些是 AutoCAD文字录入的核心基石命令，删除后将不再自动切换输入法。如遇误删，可随时通过配置面板的「恢复默认」功能进行重置。

\*\*Q3：同时打开多张图纸会互相干扰输入法状态吗？\*\*A：完全不会。v0.3.1版本已从底层重构为多文档状态隔离，每张图纸拥有独立的输入法状态机，切换文档窗口时会自动校准，互不干扰。

\*\*Q4：安装/卸载时一直提示CAD在运行，强制关闭也没用？\*\*A：这是由于AutoCAD占用了系统管理员权限，或者残留了隐形的`accoreconsole.exe`后台进程，导致标准安装包无法清理。👉v0.4.0安装包已引入“强行突破”防死锁机制：如果你确认已经手动关闭了可见的CAD窗口，只需在弹窗提示中点击“是（强行继续）”，程序将自动跨越权限阻断完成清理。

本程序提供纯净的卸载体验，不留系统垃圾：

## 🛠️ 开发与构建

以下内容面向希望参与本项目开发或自行编译的极客与开发者。

### 环境要求

- Visual Studio 2026+
- .NET Framework 2.0 ~~ 4.8 / .NET 8.0 ~~ 10.0（视目标 CAD 版本而定）

### 项目结构

```
CAD Auto IME/            
├── build/                # 编译输出目录            
├── installer/            # 安装包项目 (Inno Setup)            
├── lib/                  # AutoCAD SDK 依赖 (CAD DLLs)            
│   ├── R17.0-R17.2_CAD_2007-2009/            
│   ├── R18.0_R18.2_CAD_2010-2012/            
│   ... (其他版本)            
│   └── R26.0_CAD_2027/            
├── OpenCadIme.Core/      # 核心逻辑共享项目 (Shared Project)            
│   ├── Core/             # 核心逻辑 (CommandInterceptor, ConfigManager 等)            
│   ├── Interop/          # Win32 API 调用            
│   ├── UI/               # UI 逻辑 (HudManager, LegacyForm, ModernWpf)            
│   ├── AppConstants.cs   # 常量定义            
│   ├── PluginMain.cs     # 插件入口            
│   └── UpdateManager.cs  # 更新检查            
├── src/                  # 各版本适配层 (版本差异化包装)            
│   ├── OpenCadIme_Sys17/            
│   ├── OpenCadIme_Sys18/            
│   ... (其他 SysXX 项目)            
│   └── OpenCadIme_Sys27/            
├── CAD Auto IME.sln      # Visual Studio 解决方案文件            
├── LICENSE.txt           # 开源协议            
└── README.md             # 项目说明文档
```

### 构建步骤

1. 克隆本仓库到本地。
2. 在 `lib/` 目录下的对应版本文件夹中，置入合规的 AutoCAD 依赖程序集（`acmgd.dll`, `acdbmgd.dll`, `accoremgd.dll`）。
3. 在 Visual Studio 中将构建配置切换至 **Release** 模式。
4. 选择对应的目标系统版本项目，生成解决方案即可。
