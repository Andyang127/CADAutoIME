产品概述

🔹 产品定位

CAD Auto IME 是一款专为工程设计领域打造的底层系统级效率增强组件。v0.4.0 版，跨越了 WinForms 与 WPF 的技术鸿沟，修复了极端环境下的加载死锁，为全系 AutoCAD 提供 0 崩溃的体验。

v0.4.0 版核心特性

本次版本全面攻克了长期困扰业界插件的“跨版本 API 分裂”与“启动卡死”难题：

1. 跨世代极客加载引擎 (Anti-Freeze)

引入了“多重保险监听 + 懒加载”技术。彻底解决早期 CAD (2007) 瞬间加载报错与现代 CAD (2026) 启动画面死锁的矛盾。像猎人一样耐心，只在图纸 100% 准备就绪的瞬间安全切入系统。

2. 原生句柄智能 UI 挂载

通过底层动态反射技术，智能适配 2007 的 MFC 消息循环与 2026 的 WPF/XAML 新架构。独家研发 `AssemblyResolve` 导航探针，让极其挑剔的高版本 CAD 也能顺滑弹出极其现代的交互面板，杜绝一切静默崩溃。

3. 双擎状态锁：0 误杀防御网

我们将"命令状态"与"焦点弹窗"深度结合。只要你执行的不是白名单里的文字命令，底层就会挂上一把"死锁"，即使界面弹出一万个下拉菜单抢占焦点，插件也绝对不会切出中文。

4. 全透明双区平权 UX 与无感交互

彻底抛弃"黑盒"逻辑。现在的配置面板中，所有 AutoCAD 核心文字命令（如 TEXT, MTEXT, BLOCK, LAYER 等 28 个）与外挂命令完全透明可见。配合智能防打扰判断（只在后台加载时弹窗，手工敲命令不再遮挡视线），交互如丝般顺滑。

5. 攻克隐式命令（双击修改 _TEXTEDIT）

解决了以往双击单行文字进行修改时，因底层触发隐式 _TEXTEDIT 等特殊命令而导致无法自动切换输入法的顽疾。现在无论是敲击快捷键，还是鼠标双击直接编辑，中文输入法都能精准响应。

使用指南与命令

极简操作提示：本程序的当前版本自带 97 个 内置命令，包括 46 个 CAD 原生系统命令和 51 个 常用插件命令。可以直接上手画图，如发现有个别自定 LISP 没有切中文，再手动添加即可。
添加自定义命令的铁律：必须输入“命令全称”！
在配置面板中添加扩展命令时，严禁输入快捷键（Alias）（例如画圆不要输 C，创建块不要输 B）。你必须输入 CAD 底部命令行执行时的真实全称（例如：CIRCLE、BLOCK、TBLKNAME 等），否则系统底层将无法匹配触发！

🔹 1. 呼出白名单配置面板

在 CAD 命令行输入该命令，即可直观管理所有文字、标注指令。支持 Ctrl+Z 撤销，Ctrl+Y 重做，以及双击/Delete 一键防呆删除。

COMMAND: CUSTOMAUTOIME

🔹 2. 临时开启 / 关闭助手

在 CAD 命令行输入该命令，即可一键全局切换插件的运行状态（开启/关闭监控）。

COMMAND: TOGGLEAUTOIME

常见问题解答

Q1：我可以把面板下方的 TEXT、MTEXT 等系统内置指令删掉吗？

A：强烈建议保留！它们是维持 AutoCAD 正常文字录入的核心骨干。虽然我们将控制权完全交给了你（允许你强行删除），但删除后，这些命令将不再自动切换输入法。如果误删，请点击面板下方的【恢复默认】按钮一键重置。

Q2：程序会影响 AutoCAD 运行速度、引起掉帧卡顿吗？

A：绝对不会。本版本彻底消灭了高频拦截带来的垃圾回收（GC）压力，通过底层内存 Buffer 复用机制，对 CPU 和帧率的影响严格为 0。哪怕连续画图 10 小时，鼠标依然顺滑如初。

Q3：同时打开多张图纸会互相影响吗？

A：已经彻底解决此问题。每张图纸拥有独立的输入法状态机，完全物理隔离。在图纸 A 中输入文字切到中文，切到图纸 B 画线时自动恢复英文，互不干扰。

Q4：为什么安装/卸载时一直提示 CAD 在运行，点击强制关闭也没用？

A：这是因为你的 AutoCAD 占用了系统管理员权限，或者残留了隐形的 accoreconsole.exe 进程，导致标准安装包无法强杀。v0.4.0 安装包已引入“强行突破”防死锁机制，如果你确认已经手动关闭了 CAD，只需在弹窗提示中点击“是（强行继续）”，即可无视系统权限拦截，完美完成安装与卸载。

卸载流程

本程序提供极其纯净的卸载体验，不留任何系统垃圾。

  1. 打开 Windows 「开始菜单」，找到 「CAD Auto Ime」 程序组。
  2. 点击 「卸载 CAD 输入法助手」 选项。
  3. 注意：为确保彻底清除注册表占用，卸载时若 CAD 正在运行，程序会弹出友好提示建议您保存图纸并协助自动关闭 CAD 进程。遇权限锁定时可选择强行跳过。
